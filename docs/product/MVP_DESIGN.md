# 攒股收息MVP需求分析

## 主线一：历史回测

核心问题：

> 过去按固定金额持续买入某个标的或组合，结果怎么样？

MVP包含：

- A股股票、ETF；
- 单标的回测；
- 多标的相同条件并排比较；
- 简单组合回测；
- 每月固定金额投入；
- 指定买入日，非交易日顺延；
- 按实际一手买入；
- 各标的资金独立结转；
- 分红回购原标的；
- 剩余现金参考历史国债逆回购收益；
- 简化费用和精确费用；
- 固定区间、快捷区间、滚动区间。

核心输出只保留：

1. 累计投入；
2. 最终资产；
3. 累计收益率；
4. 年化收益率；
5. 最大回撤；
6. 最长亏损时间；
7. 累计分红；
8. 国债逆回购收益；
9. 分红稳定性；
10. 历史估值参考。

回测结果可以保存为“参考方案”，但 MVP 不做复杂计划管理。

---

## 主线二：实际记录

核心问题：

> 我实际买了什么、投入了多少钱、现在赚了多少？

用户手工记录：

- 资金转入；
- 股票或ETF买入；
- 股票或ETF卖出；
- 国债逆回购；
- 资金转出；
- 必要的手工调整。

买入只需输入：

- 标的；
- 日期；
- 买入价格；
- 手数；
- 实际费用可选修改。

国债逆回购记录：

- 品种；
- 本金；
- 成交收益率；
- 日期和期限；
- 手续费；
- 到期收益。

系统自动补充：

- 标的名称和行情；
- 当前市值；
- 历史分红信息；
- 预计分红；
- 登记日持股数量；
- 国债逆回购预计收益。

用户可以修改自动数据，最终以用户确认值为准。

实际记录核心输出：

1. 累计资金转入；
2. 累计资金转出；
3. 净投入；
4. 当前股票及ETF市值；
5. 可用现金；
6. 逆回购本金；
7. 累计分红；
8. 累计逆回购收益；
9. 当前总资产；
10. 累计收益率；
11. 年化收益率。

---

# 两条主线之间的关系

```text
历史回测
    ↓
选出准备长期积累的标的或组合
    ↓
实际买入并手工记录
    ↓
自动更新行情、分红和收益
    ↓
对照历史回测结果
```

不过 MVP 中，“回测与实盘对照”只需要提供一个轻量入口，不需要建立复杂的执行偏差分析系统。

# MVP页面可以进一步简化

底部只保留四个入口：

| 页面 | 用途                           |
| ---- | ------------------------------ |
| 回测 | 创建回测、查看比较结果         |
| 持有 | 当前资产、持仓、现金与收益     |
| 收息 | 分红日历、预计分红、已到账分红 |
| 我的 | 数据源、费用、备份与设置       |

右下角“+”统一用于记录：

- 资金转入；
- 买入；
- 卖出；
- 国债逆回购；
- 资金转出；
- 手工调整。

不再单独设置：

- 标的研究页；
- 定投计划页；
- 退休领取页；
- FIRE进度页；
- 独立账本页；
- 组合优化页。

## MVP的一句话定义

> **“攒股收息”通过历史回测帮助用户选择A股股票和ETF，并通过手工记录与自动行情补充，持续计算真实资产、分红和综合收益。**

现在需要确认的只是：你说的两条主线，是否就是 **“历史回测”和“实际记录”**。
