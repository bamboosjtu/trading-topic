## 1. 引言：为什么需要金融数据包

在量化交易的 pipeline 中，数据是地基。无论回测策略还是实盘决策，第一步永远是**获取可靠的历史与实时数据**。手动从网页爬取数据不仅低效，还面临反爬、格式变动、鉴权等问题。

金融数据包（Financial Data Packages）封装了数据获取的复杂度，让你用 **几行 Python 代码** 拿到结构化的 DataFrame。本指导书选取了四个最具代表性的包：

| 包 | 数据市场 | 核心定位 |
|---|---|---|
| **akshare** | A股/期货/宏观/基金/债券 | 免费聚合器，覆盖面最广 |
| **tushare** | A股/港股/美股/宏观/另类 | 专业标准化，Token 鉴权 |
| **ccxt** | 加密货币（100+交易所） | 统一交易接口，跨所套利 |
| **yfinance** | 全球股票/ETF/外汇/商品 | 轻量级，雅虎数据直达 |

## 3. akshare — 国内免费数据聚合器

### 3.1 设计理念

> **"一个函数，一个数据源"** —— 把国内主流财经网站（东方财富、新浪、同花顺、巨潮资讯等）的公开数据接口封装成统一风格的 Python 函数。

akshare 的核心哲学：

- **零配置**：无需注册、Token、API Key，`pip install` 即可用
- **源头透明**：函数名直接反映数据来源（`stock_zh_a_hist` = 东方财富A股历史行情）
- **覆盖面广**：A股/港股/期货/期权/基金/债券/宏观/另类数据，几乎涵盖所有国内公开金融数据
- **时效优先**：直接调用源头 HTTP API，不做中间缓存（除非源头本身有缓存）

```
┌──────────────────────────────────┐
│            akshare                │
│  ┌─────────────────────────────┐ │
│  │  统一函数签名  →  DataFrame  │ │
│  └─────────────────────────────┘ │
│        ↑         ↑        ↑      │
│   ┌────┴───┐ ┌──┴──┐ ┌───┴───┐  │
│   │东方财富 │ │新浪 │ │巨潮资讯│  │
│   └────────┘ └────┘ └───────┘  │
│   (HTTP API，无需鉴权)           │
└──────────────────────────────────┘
```

### 3.2 核心函数速览

```python
import akshare as ak

# ──── 行情类 ────
# A股个股历史日K线
df = ak.stock_zh_a_hist(symbol="000001", period="daily", start_date="20240101", end_date="20240701")

# A股实时行情（全市场快照）
df = ak.stock_zh_a_spot_em()

# 指数历史数据
df = ak.stock_zh_index_daily(symbol="sh000001")  # 上证指数

# ──── 基本面类 ────
# 个股财务指标
df = ak.stock_financial_abstract(symbol="000001")

# ──── 另类数据 ────
# 北向资金
df = ak.stock_hsgt_north_net_flow_in_em(symbol="北上")

# 融资融券
df = ak.stock_margin_detail_sse(date="20240630")
```

### 3.3 适用场景

- ✅ A股策略研究与回测数据准备
- ✅ 需要快速验证一个想法（"茅台历史上股息率最高是多少？"）
- ✅ 爬取东方财富/新浪上能看到的任何公开数据
- ❌ 需要毫秒级实时行情（这是聚合器，不是交易所直连）
- ❌ 需要完整的历史深度（比如二十年前的退市股数据）

## 4. tushare — 专业级金融数据平台

### 4.1 设计理念

> **"数据即服务"** —— 建立标准化的数据模型，统一 API 规范，通过 Token 机制实现分级授权。

tushare 与 akshare 的根本不同在于：**它不是爬虫，而是一个数据服务平台**。背后有独立的数据库和清洗流水线。

核心哲学：

- **标准化**：所有接口返回结构一致的 DataFrame，字段名、类型统一
- **分级授权**：基础数据免费（需注册），高频/深度数据需积分兑换
- **数据清洗**：服务端做过去重、复权校准、停牌标记等预处理
- **PRO 接口体系**：新版接口统一以 `pro_api()` 方式调用，参数通过 dict 传入

```
┌──────────────────────────────────┐
│            tushare               │
│  ┌─────────────────────────────┐ │
│  │   pro_api() 统一入口         │ │
│  │   参数 → JSON → 服务端查询   │ │
│  └─────────────────────────────┘ │
│          ↓ (Token 鉴权)          │
│  ┌─────────────────────────────┐ │
│  │      Tushare 后端数据库       │ │
│  │  (独立采集 → 清洗 → 存储)     │ │
│  └─────────────────────────────┘ │
└──────────────────────────────────┘
```

### 4.2 Token 注册

1. 访问 https://tushare.pro → 注册账号
2. 登录后 → 个人主页 → "接口TOKEN" → 复制
3. 设置 Token：

```python
import tushare as ts
ts.set_token("你的TOKEN")
pro = ts.pro_api()
```

### 4.3 核心接口速览

```python
import tushare as ts

ts.set_token("YOUR_TOKEN")
pro = ts.pro_api()

# ──── 行情类 ────
# 日线行情
df = pro.daily(ts_code="000001.SZ", start_date="20240101", end_date="20240701")

# 分钟线
df = pro.stk_mins(ts_code="000001.SZ", freq="5min")

# ──── 基本面类 ────
# 利润表 / 资产负债表 / 现金流量表
df = pro.income(ts_code="000001.SZ", period="20231231")
df = pro.balancesheet(ts_code="000001.SZ", period="20231231")
df = pro.cashflow(ts_code="000001.SZ", period="20231231")

# 财务指标（ROE/ROA/毛利率等）
df = pro.fina_indicator(ts_code="000001.SZ")

# ──── 特色数据 ────
# 沪深股通十大成交股
df = pro.hsgt_top10(trade_date="20240701")

# 龙虎榜
df = pro.top_list(trade_date="20240701")

# 限售解禁
df = pro.share_float(ts_code="000001.SZ")
```

### 4.4 适用场景

- ✅ 需要长期维护的量化系统（接口稳定性好）
- ✅ 需要三张财务报表 + 财务指标的统一数据
- ✅ 与团队协作（Token 统一管理，数据格式标准）
- ❌ 快速原型验证（注册+Token 有门槛，不如 akshare 即装即用）

## 5. ccxt — 加密货币交易所统一接口

### 5.1 设计理念

> **"一个 API，所有交易所"** —— 将 100+ 加密货币交易所的 REST/WebSocket API 抽象为一套统一的方法签名。

ccxt 的核心哲学：

- **统一方法名**：不管底层是 Binance 还是 OKX，`fetch_ohlcv()` 就是拿K线，`create_order()` 就是下单
- **标准化输出**：所有交易所返回的 OHLCV 数据格式完全相同，方便跨所策略
- **市场无关**：你写的套利策略可以不加修改地跑在 Binance↔OKX 之间
- **社区驱动**：新交易所、新功能的支持靠社区贡献（GitHub 60000+ Star）

```
┌──────────────────────────────────────┐
│               ccxt                   │
│  ┌────────────────────────────────┐  │
│  │   统一方法                     │  │
│  │  fetch_ohlcv()                 │  │
│  │  fetch_ticker()                │  │
│  │  create_order()     ...        │  │
│  └────────────────────────────────┘  │
│        ↓  (按 exchange id 分发)       │
│  ┌──────┐ ┌──────┐ ┌──────┐         │
│  │Binance│ │ OKX  │ │Bybit │  ...    │
│  └──────┘ └──────┘ └──────┘         │
│   (REST + WebSocket 连接池)          │
└──────────────────────────────────────┘
```

### 5.2 核心方法速览

```python
import ccxt

# 创建交易所实例
exchange = ccxt.binance({
    'enableRateLimit': True,   # 内置限流，防止被封
    'options': {'defaultType': 'spot'},  # 现货
})

# 不需要 API Key 的公开接口：
# ──── 市场数据 ────
markets = exchange.load_markets()                              # 加载所有交易对
ticker  = exchange.fetch_ticker('BTC/USDT')                    # 实时行情
ohlcv   = exchange.fetch_ohlcv('BTC/USDT', '1d', limit=100)   # K线
orderbook = exchange.fetch_order_book('BTC/USDT', limit=20)   # 订单簿深度
trades  = exchange.fetch_trades('BTC/USDT', limit=50)          # 最近成交
```

### 5.3 适用场景

- ✅ 加密货币量化交易（现货/合约）
- ✅ 跨交易所套利（"搬砖"）
- ✅ 做市策略开发
- ❌ 传统股票/期货市场（ccxt 专注加密货币）

## 6. yfinance — 雅虎财经全球市场数据

### 6.1 设计理念

> **"极简即用"** —— 用 `yf.download()` 一行代码拿到全球任何市场的历史数据。

yfinance 的设计极简到了极致：

- **一行下载**：`yf.download('AAPL')` 即可获取苹果全部历史
- **全球化**：覆盖美股/港股/A股/外汇/期货/ETF/指数/加密货币
- **零配置**：无需 API Key，直接访问雅虎财经公开接口
- **Ticker 对象模型**：每个标的物是一个对象，`.history()` `.info` `.dividends` 方法自然派生

```
┌──────────────────────────────┐
│          yfinance             │
│  ┌─────────────────────────┐ │
│  │  Ticker("AAPL")          │ │
│  │   .history()  → K线      │ │
│  │   .info       → 基本面   │ │
│  │   .dividends  → 分红     │ │
│  │   .financials → 财报     │ │
│  └─────────────────────────┘ │
│             ↓                 │
│  ┌─────────────────────────┐ │
│  │    Yahoo Finance API      │ │
│  │  (全球市场，免费公开)     │ │
│  └─────────────────────────┘ │
└──────────────────────────────┘
```

### 6.2 Ticker 对象详解

```python
import yfinance as yf

# 创建 Ticker 对象
aapl = yf.Ticker("AAPL")

# 基本信息（市值/PE/行业/描述等，一个 dict 包含上百个字段）
info = aapl.info
print(info.get("marketCap"), info.get("trailingPE"), info.get("sector"))

# 历史K线
hist = aapl.history(period="1y")           # 最近1年
hist = aapl.history(start="2020-01-01")    # 指定起始日
hist = aapl.history(period="5d", interval="1h")  # 小时线

# 分红与拆股
print(aapl.dividends.tail(10))   # 历史分红
print(aapl.splits)               # 历史拆股

# 财务报表
print(aapl.financials)           # 年度利润表
print(aapl.balance_sheet)        # 资产负债表
print(aapl.cashflow)             # 现金流量表

# 多标的同时下载
df = yf.download(["AAPL", "GOOGL", "MSFT"], start="2024-01-01")
```

### 6.3 适用场景

- ✅ 全球市场（特别是美股）的历史数据获取
- ✅ 快速查看一只股票的基本面（`.info` 非常丰富）
- ✅ 投资组合分析（多标的批量下载）
- ❌ 需要 A 股 Level-2 盘口/逐笔成交（yfinance 不支持）
- ❌ 中国大陆网络环境不稳定时（可能需要代理）

## 7. 四包设计理念对比

| 维度 | akshare | tushare | ccxt | yfinance |
|---|---|---|---|---|
| **设计模式** | 函数式聚合器 | 服务化数据平台 | 交易所适配器 | 对象化数据下载器 |
| **核心抽象** | 一个函数 = 一个数据源 | pro_api + 统一参数 | exchange.fetch_* | Ticker 对象 |
| **鉴权方式** | 零鉴权 | Token 分级 | 可选 API Key | 零鉴权 |
| **数据市场** | A股/期货/宏观/基金 | A股/港股/美股/宏观 | 加密货币 | 全球股票/ETF/外汇 |
| **数据来源** | 爬取公开网站 | 自有数据库 | 交易所 API 直连 | Yahoo Finance |
| **数据深度** | 看源头能提供什么 | 标准化、深度清洗 | 看交易所提供什么 | 雅虎覆盖范围 |
| **实时性** | 取决于源头刷新频率 | T+1（部分延迟15min） | 毫秒级（WebSocket） | 延迟 15 分钟 |
| **稳定性** | 源头变动可能失效 | 高（自有数据库） | 高（交易所官方 API） | 中（雅虎接口偶有变动） |
| **学习曲线** | 低（纯函数调用） | 中（需注册+查文档） | 中（需理解交易所概念） | 极低 |
| **社区规模** | 国内量化圈主流 | 国内机构主流 | GitHub 64k+ Star | GitHub 15k+ Star |

## 8. 实验一：akshare 基础实验

### 实验目的
掌握 akshare 的基本用法：获取 A 股历史行情、实时快照、北向资金，理解"一个函数一个数据源"的设计模式。

### 实验脚本

```python
"""
实验一：akshare 基础实验
目标：使用 akshare 获取 A 股行情、基本面、另类数据
"""

import akshare as ak
import pandas as pd
import matplotlib.pyplot as plt

# 设置中文显示
plt.rcParams["font.sans-serif"] = ["SimHei", "Microsoft YaHei"]
plt.rcParams["axes.unicode_minus"] = False

# ============================================================
# 任务 1：获取 A 股历史日线数据
# ============================================================
print("=" * 60)
print("任务 1：获取平安银行(000001) 2024 年日线数据")
print("=" * 60)

df_daily = ak.stock_zh_a_hist(
    symbol="000001",
    period="daily",
    start_date="20240101",
    end_date="20240701",
    adjust="qfq"  # 前复权
)

print(f"数据形状: {df_daily.shape}")
print(df_daily.head())
print(f"\n列名: {df_daily.columns.tolist()}")

# 画收盘价走势
df_daily["日期"] = pd.to_datetime(df_daily["日期"])
df_daily.set_index("日期", inplace=True)

fig, axes = plt.subplots(2, 1, figsize=(12, 8))

axes[0].plot(df_daily.index, df_daily["收盘"], color="#2196F3", linewidth=1.2)
axes[0].set_title("平安银行(000001) 2024年H1 收盘价", fontsize=14)
axes[0].set_ylabel("价格 (元)")
axes[0].grid(True, alpha=0.3)

axes[1].bar(df_daily.index, df_daily["成交量"], color="#FF9800", alpha=0.6, width=0.8)
axes[1].set_title("成交量", fontsize=14)
axes[1].set_ylabel("成交量 (手)")
axes[1].grid(True, alpha=0.3)

plt.tight_layout()
plt.savefig("lab1_task1_kline.png", dpi=150)
plt.close()
print("→ 图表已保存: lab1_task1_kline.png")

# ============================================================
# 任务 2：获取全市场实时行情快照
# ============================================================
print("\n" + "=" * 60)
print("任务 2：A股全市场实时行情快照")
print("=" * 60)

df_spot = ak.stock_zh_a_spot_em()

print(f"全市场股票数: {len(df_spot)}")
print(f"列名: {df_spot.columns.tolist()[:10]}...")

# 按成交额排序 TOP10
if "成交额" in df_spot.columns:
    top10_amount = df_spot.nlargest(10, "成交额")[["代码", "名称", "最新价", "涨跌幅", "成交额", "换手率"]]
    print("\n📊 成交额 TOP10：")
    print(top10_amount.to_string(index=False))

# 涨停股数量
if "涨跌幅" in df_spot.columns:
    limit_up = df_spot[df_spot["涨跌幅"] >= 9.8]
    limit_down = df_spot[df_spot["涨跌幅"] <= -9.8]
    print(f"\n涨停家数(≥9.8%): {len(limit_up)}")
    print(f"跌停家数(≤-9.8%): {len(limit_down)}")

# ============================================================
# 任务 3：北向资金流向
# ============================================================
print("\n" + "=" * 60)
print("任务 3：北向资金历史流向")
print("=" * 60)

df_north = ak.stock_hsgt_north_net_flow_in_em(symbol="北上")
print(f"数据形状: {df_north.shape}")
print(df_north.tail(10))

# 画北向资金累计图
if "value" in df_north.columns or "数值" in df_north.columns:
    value_col = "value" if "value" in df_north.columns else "数值"
    date_col = "date" if "date" in df_north.columns else "日期"

    df_north["date"] = pd.to_datetime(df_north[date_col])
    df_north = df_north.sort_values("date")
    df_north["cumsum"] = df_north[value_col].cumsum()

    fig, ax = plt.subplots(figsize=(12, 4))
    ax.bar(df_north["date"], df_north[value_col],
           color=["#4CAF50" if v > 0 else "#F44336" for v in df_north[value_col]],
           alpha=0.6)
    ax2 = ax.twinx()
    ax2.plot(df_north["date"], df_north["cumsum"], color="#2196F3", linewidth=1.5)
    ax.set_title("北向资金每日净流入 & 累计净流入", fontsize=14)
    ax.set_ylabel("每日净流入 (亿元)")
    ax2.set_ylabel("累计净流入 (亿元)")
    ax.grid(True, alpha=0.3)
    plt.tight_layout()
    plt.savefig("lab1_task3_north_flow.png", dpi=150)
    plt.close()
    print("→ 图表已保存: lab1_task3_north_flow.png")

# ============================================================
# 任务 4（选做）：股票基本信息
# ============================================================
print("\n" + "=" * 60)
print("任务 4（选做）：获取个股基本信息")
print("=" * 60)

try:
    df_info = ak.stock_individual_info_em(symbol="000001")
    print(df_info.to_string(index=False))
except Exception as e:
    print(f"接口可能已变更: {e}")

print("\n✅ 实验一完成！")
```

## 12. 综合实验：跨包数据联动

### 实验目的
综合运用四个数据包，完成一个跨市场的数据分析任务：**对比 BTC 与科技股的相关性**。

### 实验脚本

```python
"""
综合实验：跨包数据联动
目标：结合 yfinance + ccxt + akshare，分析 BTC 与中美科技股的相关性
"""

import yfinance as yf
import ccxt
import akshare as ak
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns

plt.rcParams["font.sans-serif"] = ["SimHei", "Microsoft YaHei"]
plt.rcParams["axes.unicode_minus"] = False

print("=" * 60)
print("综合实验：BTC 与全球科技股相关性分析")
print("=" * 60)

# ============================================================
# Step 1: 用 yfinance 获取美股科技股
# ============================================================
print("\n[1/4] yfinance → 获取美股科技股数据...")
us_tech = ["AAPL", "MSFT", "NVDA", "META"]
df_us = yf.download(us_tech, start="2023-07-01")["Close"]
df_us_returns = df_us.pct_change().dropna()
print(f"  美股数据: {df_us_returns.shape}")

# ============================================================
# Step 2: 用 ccxt 获取 BTC 价格
# ============================================================
print("[2/4] ccxt → 获取 BTC/USDT 日线...")
exchange = ccxt.binance({"enableRateLimit": True})
ohlcv = exchange.fetch_ohlcv("BTC/USDT", "1d", since=exchange.parse8601("2023-07-01T00:00:00Z"))
df_btc = pd.DataFrame(ohlcv, columns=["timestamp", "open", "high", "low", "close", "volume"])
df_btc["timestamp"] = pd.to_datetime(df_btc["timestamp"], unit="ms")
df_btc.set_index("timestamp", inplace=True)
btc_returns = df_btc["close"].pct_change().dropna()
print(f"  BTC数据: {len(btc_returns)} 条日线")

# ============================================================
# Step 3: 用 akshare 获取 A 股半导体/科技股
# ============================================================
print("[3/4] akshare → 获取A股科技股数据...")
cn_tech_codes = ["002371", "688981", "603501", "002049"]  # 北方华创/中芯国际/韦尔股份/紫光国微
cn_tech_names = ["北方华创", "中芯国际", "韦尔股份", "紫光国微"]
cn_data = {}

for code, name in zip(cn_tech_codes, cn_tech_names):
    try:
        df = ak.stock_zh_a_hist(symbol=code, period="daily",
                                 start_date="20230701", end_date="20250701",
                                 adjust="qfq")
        df["日期"] = pd.to_datetime(df["日期"])
        df.set_index("日期", inplace=True)
        cn_data[name] = df["收盘"]
        print(f"  {name}({code}): {len(df)} 条")
    except Exception as e:
        print(f"  {name}({code}): 获取失败 - {e}")

df_cn = pd.DataFrame(cn_data)
df_cn_returns = df_cn.pct_change().dropna()
print(f"  A股数据: {df_cn_returns.shape}")

# ============================================================
# Step 4: 对齐日期 & 计算相关性
# ============================================================
print("[4/4] 对齐日期 & 计算相关性矩阵...")

# 合并所有收益率数据
all_returns = df_us_returns.copy()
all_returns["BTC"] = btc_returns
for col in df_cn_returns.columns:
    all_returns[col] = df_cn_returns[col]

# 删除 NaN 并对齐
all_returns = all_returns.dropna()
print(f"  对齐后数据: {all_returns.shape}")

# 相关性矩阵
corr_matrix = all_returns.corr()

# 画图
fig, axes = plt.subplots(1, 2, figsize=(16, 7))

# 子图1：相关性热力图
mask = np.triu(np.ones_like(corr_matrix, dtype=bool), k=1)
sns.heatmap(corr_matrix, mask=mask, annot=True, fmt=".2f",
            cmap="RdBu_r", center=0, vmin=-1, vmax=1,
            square=True, linewidths=0.5, ax=axes[0],
            cbar_kws={"shrink": 0.8})
axes[0].set_title("BTC · 美股科技 · A股科技 收益率相关性", fontsize=14)

# 子图2：BTC vs 各标的 滚动相关性
rolling_window = 60
btc_roll = all_returns["BTC"].rolling(rolling_window)
fig2_data = {}

for col in ["AAPL", "NVDA", "中芯国际"]:
    if col in all_returns.columns:
        roll_corr = all_returns[col].rolling(rolling_window).corr(all_returns["BTC"])
        axes[1].plot(roll_corr.index, roll_corr, linewidth=1.2, label=f"BTC vs {col}")
        fig2_data[col] = roll_corr

axes[1].axhline(y=0, color="black", linewidth=0.5)
axes[1].set_title(f"BTC 与主要科技股 {rolling_window}日滚动相关性", fontsize=14)
axes[1].legend(loc="lower left")
axes[1].set_ylabel("Pearson 相关系数")
axes[1].grid(True, alpha=0.3)

plt.tight_layout()
plt.savefig("comprehensive_correlation.png", dpi=150)
plt.close()
print("\n→ 图表已保存: comprehensive_correlation.png")

# ============================================================
# 结论输出
# ============================================================
print("\n" + "=" * 60)
print("分析结论")
print("=" * 60)

# 提取 BTC 与各标的的相关系数
btc_corr = corr_matrix["BTC"].drop("BTC").sort_values(ascending=False)
print("\nBTC 与各标的相关系数排名:")
for asset, corr in btc_corr.items():
    tag = "强正相关" if corr > 0.5 else ("弱正相关" if corr > 0 else ("弱负相关" if corr > -0.5 else "强负相关"))
    print(f"  {asset:>10s}: {corr:>+6.3f}  ({tag})")

print("\n💡 思考题：")
print("  1. BTC 与美股科技股的相关性是否高于与A股科技股？可能的原因是什么？")
print("  2. 滚动相关性在哪些时间段出现显著变化？对应什么市场事件？")
print("  3. 如果 BTC 与科技股高度相关，它还能充当分散化工具吗？")

print("\n✅ 综合实验完成！")
```

## 13. 常见问题与排错

### akshare

| 问题 | 可能原因 | 解决方案 |
|---|---|---|
| `ConnectionError` | 源头网站改版/下线 | 升级 akshare 到最新版：`pip install -U akshare` |
| 返回空 DataFrame | 股票代码格式不对 | 检查代码：深交所 `"000001"`，上交所 `"600000"` |
| 部分接口很慢 | 源头服务器响应慢 | 加上 try/except + 重试逻辑 |

### tushare

| 问题 | 可能原因 | 解决方案 |
|---|---|---|
| `Wrong Token` | Token 未设置或过期 | 重新从 tushare.pro 个人主页获取 |
| 接口返回权限不足 | 积分不够 | 查看 tushare.pro → 积分规则，做任务提升积分 |
| 数据量太大被限 | 频率限制 | 加 `time.sleep(0.5)` 在循环中 |

### ccxt

| 问题 | 可能原因 | 解决方案 |
|---|---|---|
| `NetworkError` | 交易所屏蔽（国内 IP） | 使用代理：`ccxt.binance({"proxies": {...}})` |
| `RateLimitExceeded` | 请求过快 | 设置 `enableRateLimit: True` |
| 某些交易所无数据 | 交易所 API 调整 | 查看 ccxt GitHub Issues |

### yfinance

| 问题 | 可能原因 | 解决方案 |
|---|---|---|
| 返回空 DataFrame | 中国大陆网络限制 | 配置代理：`yf.download(..., proxy="http://127.0.0.1:7890")` |
| `info` 为空 | 雅虎接口变动 | 升级 yfinance；或用 `fast_info` 替代 |
| 下载卡住 | 网络超时 | 设置 `timeout` 参数或使用 `auto_adjust=False` |
