# AGENTS.md

## 适用范围

本文件适用于整个仓库。若子目录以后增加自己的 `AGENTS.md`，以距离目标文件最近的规则为准。

本项目是投资研究实验室，兼顾学习、研究与开发。默认使用中文沟通和写作；代码标识符、协议名及行业通用缩写可保留英文。

## 开始工作前

1. 先阅读根目录 `README.md`，再阅读任务所在目录的 README、研究说明或产品文档。
2. 检查 `git status --short`，保留用户已有改动，不清理、不覆盖无关文件。
3. 明确任务属于学习实验、可复现研究、最终成稿还是产品开发，并遵守对应目录边界。
4. 涉及投资结论时，先确认证券、市场、币种、报告期、数据截止日和复权/收益口径。

## 目录职责

| 路径 | 职责 | 约束 |
| --- | --- | --- |
| `labs/` | Lab 1～5 研究主线、Notebook、探索性代码 | 以 `labs/Lab1-*.md` 至 `labs/Lab5-*.md` 为阶段主线 |
| `labs/Lab1-*.md`～`labs/Lab5-*.md` | 实验目标、步骤与验收口径 | 实验语义变化时同步更新对应主线文档 |
| `labs/data/` | Lab 的原始数据和处理中间数据 | 大文件、可再生文件和含授权限制的数据默认不提交 |
| `research/<topic>/` | 一个主题的可复现研究闭环 | 可包含小型审计快照、代码、测试、图表和报告 |
| `reports/` | 面向读者的最终研究成稿 | 不放通用代码、缓存和临时数据 |
| `docs/product/` | 产品需求、架构和决策记录 | 文档必须包含范围、非目标和验收标准 |
| `.agents/skills/` | Codex 投资研究工作流 | 使用相关 skill 前完整阅读其 `SKILL.md` |
| `.claude/skills/` | Claude 对应工作流 | 不假设与 `.agents/skills/` 自动同步 |

不要重新创建含义模糊的根目录 `report/`。可复现研究放入 `research/<topic>/`，最终文章放入 `reports/`。

## 研究质量底线

### 事实与来源

- 优先使用公司公告、交易所、监管机构、指数公司和政府统计等一手来源。
- 所有会随时间变化的事实都标明数据截止日；报告期与发布日期不得混用。
- 报告中的关键数字应能回溯到来源、字段、转换方法或计算代码。
- 找不到可靠证据时明确写“尚未验证”，不得补造数字、引文、来源或管理层观点。
- 严格区分事实、计算结果、假设、情景和主观判断。
- 使用二手资料时保留原始链接，并说明其用途和局限。

### 金融与回测口径

- 明确价格是否前复权、后复权或不复权，收益是否含分红，基准是否为全收益指数。
- 明确币种、单位、税费、手续费、交易日顺延、最小交易单位、现金收益和分红再投资规则。
- 避免前视偏差、幸存者偏差和时间错配；基本面数据只能在当时可获得之后使用。
- 回测结论必须同时报告收益与风险，至少包含样本区间、基准、最大回撤和主要局限。
- 不把历史回测写成收益承诺，不把相关性写成因果关系。
- 若业务口径与实现不一致，先修正语义和归属边界，不能以“代码运行成功”代替研究正确。

### 报告表达

- 结论先行，但结论强度不得超过证据强度。
- 对重大结论给出反证、风险、失效条件或待验证问题。
- 图表必须有标题、时间范围、单位、口径和来源；正文中的数字应与表格、图表和代码输出一致。
- 面向外部读者的研究成稿写入 `reports/`；同一主题的中间过程留在对应 `research/<topic>/`。

## 数据与安全

- `.env`、Token、Cookie、账号和个人信息不得提交、打印到日志或写入报告。
- 只有任务确实需要时才读取环境变量；错误信息中也要避免泄露密钥。
- `labs/data/` 中的实验数据默认由 `.gitignore` 排除。确需审计的小型快照可放入对应研究包并显式提交。
- 不覆盖已经用于已发布结论的数据快照。刷新数据时保留获取时间、来源和变更说明，必要时使用带日期或时间戳的新快照。
- 不在未经用户明确授权的情况下发送消息、发布报告、下单、连接券商或执行任何实盘操作。

## 开发与验证

Python 环境统一由 `labs/pyproject.toml` 与 `labs/uv.lock` 管理。默认从仓库根目录运行：

```powershell
uv sync --project labs
uv run --project labs python <脚本路径>
```

当前银行股定投研究的基础验证命令：

```powershell
uv run --project labs python research/bank-dca/test_analysis.py
uv run --project labs python research/bank-dca/verify_returns.py
uv run --project labs python research/bank-dca/build_report.py
```

修改代码时：

1. 优先修复根因，避免在报告中手工覆盖错误输出。
2. 将纯计算与数据获取、绘图和写文件分离，便于测试。
3. 为金融口径和边界条件增加小型确定性测试。
4. 不让项目依赖被忽略的 `quant-for-beginners/`、本地虚拟环境或机器绝对路径。
5. 目录移动后同步修正文档、命令、导入和产物路径。
6. 不随意更新锁文件；只有依赖确实变化时才运行依赖解析。

## 使用仓库 skills

任务明确匹配 `.agents/skills/<name>/SKILL.md` 时，应使用最小且相关的 skill 组合，并遵守其中的数据验证与输出要求。Skill 的报告路径若指向 `reports/`，继续沿用该约定。

Skills 是工作流，不是证据来源。即使 skill 给出了分析框架，事实与数字仍须按本文件的研究质量规则独立核验。

## 完成标准

交付前至少确认：

- 文件位于正确目录，README 或索引可找到它；
- 没有遗留旧路径、机器绝对路径、密钥或无意提交的大型生成文件；
- 相关测试、校验或最小运行命令已经执行；
- 代码输出、图表和报告关键数字一致；
- `git diff` 只包含任务范围内的变更；
- 最终说明列出变更、验证结果及仍未消除的风险。
